# Repository Analysis Guidelines

This file provides constraints and guidance for the repository analysis phase.

## Analysis Constraints

**Scope Limitations:**

- Focus on core architecture and implementation
- Prioritize novel contributions and unique approaches
- Avoid excessive detail on standard frameworks
- Emphasize academic relevance and technical depth

**Technical Focus Areas:**

- Multi-agent systems and AI architectures
- Evaluation frameworks and metrics
- Technology stack decisions and rationale
- Integration patterns and design principles

## Content Extraction Priorities

**High Priority:**

1. Unique architectural patterns
2. Novel algorithmic approaches
3. Evaluation methodologies
4. Technical innovations
5. Integration strategies

**Medium Priority:**

1. Standard framework usage
2. Conventional design patterns
3. Common development practices
4. Standard dependencies

**Low Priority:**

1. Boilerplate code
2. Standard configurations
3. Basic utility functions
4. Common setup procedures

## Academic Perspective

**Research Relevance:**

- Identify contributions to the field
- Document experimental approaches
- Highlight evaluation methodologies
- Emphasize reproducibility aspects

**Technical Rigor:**

- Validate architectural claims
- Document design decisions rationale
- Identify performance considerations
- Note scalability implications

## Output Quality Standards

- Technical accuracy and precision
- Academic-appropriate language
- Clear documentation of sources
- Proper attribution of external components
- Emphasis on novel contributions
